## 高频修复模式

当这5类问题出现时，以下修复模式经过实战验证最有效：

### 模式A：加内心独白（应对V9）
```
问题：角色人设转变无过渡
修复：在转变前加一章该角色的私人场景（深夜/独处/写信）
效果：读者看到转变的内在逻辑，不觉得突兀
实战案例：ch_20b「梁秋润的深夜」、ch_29b「两姐妹的深夜」
```

### 模式B：埋伏笔（应对V8/V14）
```
问题：重要配角空降
修复：在登场前10-20章埋2-3处暗线伏笔，每处不超过50字
效果：配角登场时读者觉得「原来如此」，不是「这人谁？」
实战案例：ch_foreshadow-lin「林叔三处伏笔」
```

### 模式C：连续执行模式（应对所有问题）
```
问题：修改过程中发现新问题
修复：不要停下询问用户，在fix-log的「新增发现」区记录后继续改
效果：一次会话解决全部问题，用户不需要反复批准
实战案例：本会话一次改完14个问题，用户只说了「继续」「要」「继续改」
```
