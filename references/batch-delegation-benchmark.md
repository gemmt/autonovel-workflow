## 子任务上下文构造模板

```markdown
项目路径：~/novels/<name>/

风格指南核心规则（voice.md）：
- 第三人称有限视角
- 短句为主（≤15字≥50%），段落≤8行
- 对话标记用角色动作代替「他说/她说」
- 禁用：「然而」「但是」「仿佛」「似乎」「突然」「猛地」「不禁」「不由得」
- 系统提示用独立段落+前空一行
- 加粗金句约1处/3章
- 每章结尾留钩子

年代背景/时间线：<关键时间点>
硬设核心：<2-3条关键设定>

已有文件（不要覆盖）：<列表>
需要写的文件：<列表>

### 大纲内容：
<逐章大纲，每章标出 POV、核心事件、名场面>
```

## 关键教训

1. **分片 ≤10 章**：超过此数几乎必然超时
2. **超时不等于失败**：始终检查磁盘而非依赖 API 返回
3. **3 并发 × 2 轮**是处理 50+ 章小说的最优模式
4. **上下文精简**：不要在 context 中放整份 outline.md，只摘录相关章节
5. **已有文件保护**：用 `md5sum` 或 `ls -l` 时间戳在事后确认未覆盖
