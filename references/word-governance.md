# 字数治理手册（Word Count Governance）

> 参考自 InkOS 的 `draft`/`write next`/`revise` 字数治理体系
> 适用所有 autonovel-workflow Phase 2 和 Phase 3b

---

## 一、核心理念

### 保守写原则

**宁可短，不要长。** 短章可以轻松扩张，长章压缩非常痛苦。

```
短章扩展成本： 1小时（加细节/内心戏/场景描写）
长章压缩成本： 4小时+（砍内容+重连剧情+调整后续）
```

在不确定章节应有篇幅时，**默认写短**，留到 Phase 3b 再做精确治理。

### 字数不是目的，节奏才是

治理字数的**真正目标**不是让所有章节一样长，而是：
1. 避免极端差异（300字 vs 8000字）
2. 保证每章有足够的叙事密度
3. 控制读者阅读时长在合理范围（10-15分钟/章）

## 二、字数目标设定指南

### 2.1 按场景类型

| 场景类型 | 建议字数 | 典型章节 |
|:---------|:--------:|:---------|
| 日常/建立 | 2000-2500 | 角色出场、世界观交代 |
| 对话/情感 | 2500-3500 | 对手戏、内心独白、名场面 |
| 冲突/转折 | 3000-4000 | 吵架、袭击、意外、秘密揭示 |
| 过渡/喘息 | 1500-2000 | 双线间的缓冲区、插叙插笔 |
| 高潮 | 3500-5000 | 全书最重要的事件（不超过 2-3 章） |
| 插叙/伏笔 | 1000-1500 | 单独埋暗线的章节（如 foreshadow-lin） |
| 四合一/多合一 | 3500-5000 | 合并多章内容的特殊格式 |

### 2.2 按小说类型（中文网文）

| 类型 | 建议每章字数 | 说明 |
|:----|:-----------:|:-----|
| 快节奏甜文/年代文 | 2000-3000 | 读者期待快速更新，短章快切 |
| 玄幻/修仙长篇 | 2500-4000 | 世界观信息量大，需要更多篇幅 |
| 悬疑/推理 | 3000-4500 | 铺垫和揭秘需要足够空间 |
| 轻小说/日常 | 1500-2500 | 轻松阅读，不宜太长 |
| 实体书风格 | 4000-6000 | 每章如一个短篇，独立完整性 |

### 2.3 `--words` 覆盖机制

当本章是明确的名场面/高潮时，在写入前声明覆盖值：

```
请求：写第31章（梁秋润断臂）
--words 4000
```

覆盖值会写入 `state.json.word_governance.per_chapter.ch_NN.override`，覆盖默认 target 计算。

**适用于：**
- ✅ 全书最关键的 2-3 章高潮
- ✅ 需要大量篇幅的复杂双线叙事
- ✅ 用户明确指定「这章写长点」

**不适用于：**
- ❌ 日常章节（一律保守写）
- ❌ 过渡章节
- ❌ 不想思考就想写长的偷懒场景

## 三、治理执行机制

### 3.1 Phase 2：实时治理（每章发生后立即执行）

| 偏差 | 处理 | 后续 |
|:----|:-----|:-----|
| < 50% target | 暂不保存，必须补写到 ≥ 60% | 趁热补 |
| 50-70% | 保存，标记 `ok:"short"` | Phase 3b 扩张 |
| 70-130% | **目标区间，🟢** | 无后续 |
| 130-200% | 保存，标记 `ok:"overshoot"` | Phase 3b 压缩 |
| > 200% | 强警告，建议拆分或大压缩 | Phase 3b 优先处理 |

### 3.2 Phase 3b.2b：批量治理（进入深改前执行）

**顺序：**
1. 统计所有章节的 `actual` vs `target`
2. 按偏差严重度排序：🔴 > 🟡 overshoot > 🟡 short
3. 🔴 优先处理（拆章/大压缩/大扩展）
4. 🟡 统一处理（微调）
5. 输出 `revision/word-governance-report.md`
6. 更新 `state.json`

**修复案例：**

| 原文 | 问题 | 修复后 | 操作 |
|:----|:----|:-------|:-----|
| `「好的，我明白了，那我先回办公室了。」` | 拖沓 | `「我先走了。」` | 删 6 字 |
| 连续 5 段一行对话 | 凑字数 | 合并为 2 段，加行动描写 | 砍 40% |
| 大段环境描写拖慢节奏 | 填充 | 删到只剩 1-2 句 | 砍 60% |
| `他看着她，她看着他，两人都没说话。` | AI 味 | `两人对着沉默了三秒。` | 替换 |
| 内心戏和叙事混在一起 | 散漫 | 抽离为独立短段 | 重组 |

### 3.3 字数计算规则

```
zh_chars = 仅统计中文字符（去除空格/标点/英文/数字/换行）

✅ 正确示例：
  文本： "梁秋润端着茶杯" → zh_chars = 6（梁秋润端着茶杯）

❌ 错误计算：
  计入 "、" "。" "——" "「」" → 标点不算字数
  计入 "a" "1" "2024" → 英文/数字不算（除非是角色对话中的英文）
```

## 四、常见陷阱

### 🚫 陷阱 1：追求整齐划一
**症状：** 所有章节都压到 2500 字整，节奏感全失
**解法：** 允许 ±20% 的正常波动。日常章 2000、高潮章 4000 是合理的

### 🚫 陷阱 2：为了凑字数注水
**症状：** 「他看了看窗外，又看了看她，然后看了看表」→ 扩写式注水
**解法：** 字数不够时先加**有效内容**（细节/内心/氛围），不是重复描述

### 🚫 陷阱 3：拆章成风
**症状：** 6000 字的章拆成 3 个 2000 字的章，每章什么都没有推进
**解法：** 拆章的前提是每章有**独立的高潮/冲突/悬念**。拆成空壳更糟糕

### 🚫 陷阱 4：忽视标点对节奏的影响
**症状：** 字数达标了但全是逗号连接的长句
**解法：** 字数治理只看数量，节奏检测靠 voice.md 的句长分布。两者配合使用

## 五、统计与报告

### word-governance-report.md 格式

```markdown
# 字数治理报告 · 《小说名》

## 总览
- 总章节数：N
- 治理通过率：X% 🟢
- 治理后平均字数：N

## 治理前 vs 治理后

| 章节 | 治理前 | 治理后 | 目标 | 偏差变化 |
|:----|:-----:|:-----:|:----:|:--------|
| ch_NN | 980 | 1800 | 2000 | 49%→90% 🟢 |
| ch_NN | 3100 | 2600 | 2500 | 124%→104% 🟢 |

## 修复记录
- ch_XX: 扩展内心戏，+800字 → 合格
- ch_YY: 砍冗余对话，-500字 → 合格
- ch_ZZ: 拆分为两章（zz_a + zz_b）→ 双合格

## 未通过（待下轮）
- （无/列出）
```
