# 文风清理协议（Style Cleanup Protocol）

> 用于 autonovel-workflow Phase 3 执行阶段（对抗式编辑→修改简报→按优先级逐条修复）
> 实测有效：2026-06-18 对 54章/138,000字 年代文完成 P2/P3/P4 清理

---

## 概述

Phase 3a 审计报告产出后，修改简报中通常包含三类文风格式问题，按投入产出比排列：

| 优先级 | 问题 | 典型违规 | 实测数据 |
|:-----:|:-----|:---------|:--------:|
| 🔴 P1 | **加粗金句超标** | voice.md 规定全书≤15处，实测476处 | 每章平均9处 |
| 🔴 P2 | **禁用词未清零** | voice.md 规定 10+ 禁用词，实测73处 | 最严重章 ch_49(9处) |
| 🟡 P3 | **章节偏短（<50% target）** | 补充/插章普遍不足 | 6章平均仅43%目标 |

以下协议分别描述三种修复模式。

---

## 修复模式 A：加粗削减（Bold Reduction）

### 适用场景
voice.md 规定「全书加粗<1处/章，总<15处」，但初稿产出时子任务在系统提示对话中大量使用 **加重**，导致全线超标。

### 执行流程

**Step A1：摸底扫描**
```python
import re, glob
base = 'chapters/'
for f in sorted(glob.glob(base + 'ch_*.md')):
    with open(f) as fh:
        content = fh.read()
    bolds = re.findall(r'\*\*[^*]+\*\*', content)
    content_bolds = [b for b in bolds if b != '**POV**']  # 排除结构标记
    if content_bolds:
        print(f'{f}: {len(content_bolds)}处')
```
用 `execute_code` 而非 terminal（Unicode 稳定）。输出后计算总计。

**Step A2：确定保留清单（15处以内）**
从所有加粗中挑出真正的名场面。标准：
- 是全书读者会记住的句子
- 不是「系统提示」「叮——」等操作界面文本
- 不重复（同一句在不同章只保留1处）
- 覆盖所有主要角色/情感线的里程碑

**Step A3：分批次清除（推荐 delegate_task 并行）**

| 批次 | 策略 | 工具 |
|:----|:-----|:-----|
| A组（最严重章，10+处） | 3-5个文件/批，各自独立处理 | delegate_task × 2-3 |
| B组（中度 5-10处） | 5-8个文件/批 | delegate_task × 2 |
| C组（轻度 1-5处） | 批量处理 | delegate_task × 1 |

每个子任务的 context 必须包含：
- **保留清单**（明确到句子的精确文字）
- **操作指令**：对不在清单中的每一处 **...**，删除外层 ** 符号（`patch` 逐处替换），保留文字内容
- 注意：`replace_all` 会连保留条目一起删掉，不要用

**Step A4：验证**
```python
import re, glob
for f in sorted(glob.glob('chapters/ch_*.md')):
    with open(f) as fh:
        c = fh.read()
    bolds = [b for b in re.findall(r'\*\*[^*]+\*\*', c) if b != '**POV**']
    if len(bolds) > 1:
        print(f'❌ {f}: {len(bolds)}处 — {bolds}')
```

### 典型违规模式
- 系统提示面板全部加粗（「叮——检测到有效互动。计时开始。」→ 取消）
- 角色的每句内心独白都加粗（「这日子，值了。」「舒服。」→ 只保留最强1处）
- 对话中的强调语气加粗（「这是第一次，梁秋润没有推开她。」→ 保留，这是名场面）

## 修复模式 B：禁用词清理（Forbidden Word Cleanup）

### 适用场景
voice.md 明确列出若干禁用词（如：忽然、好像、忍不住、突然、猛地、然而、但是、却），初稿全篇超标。

### 执行流程

**Step B1：全景扫描**
```python
# 用 search_files(file_glob='ch_*.md', pattern='忽然|好像|...') 定位每处
```

**Step B2：逐处替换策略**

| 禁用词 | 替换方案 | 示例 |
|:-------|:---------|:-----|
| 忽然 | 直接删除字眼 | 「她忽然笑了」→「她笑了」 |
| 好像 | 删掉或改为「像」 | 「好像很开心」→「很开心」 |
| 忍不住 | 改为动作本身 | 「忍不住笑了」→「笑了出来」 |
| 突然 | 用短句换行/删掉 | 「他突然站起来」→「他站起来。」 |
| 猛地 | 直接删除 | 「猛地抬头」→「抬头」 |
| 然而/但是 | 用断句过渡 | 「然而他没有回答」→「他没有回答。」 |
| 却 | 改为「可」「但」，或断句 | 「他却没有」→「他没有。」 |

**Step B3：分批修复**
每个替换必须：
1. 用 `search_files` 定位到具体行
2. 读该行附近 3-5 行上下文
3. 用 `patch` 逐处替换（不要 replace_all——不同位置需要不同上下文替换）
4. 验证该文件搜索结果为 0

**Step B4：全量验证**
```python
import re, glob
forbidden = ['忽然','好像','忍不住','突然','猛地','然而','但是','却']
for f in sorted(glob.glob('chapters/ch_*.md')):
    with open(f) as fh:
        c = fh.read()
    hits = {w: c.count(w) for w in forbidden if c.count(w) > 0}
    if hits:
        print(f'❌ {f}: {hits}')
```

### 注意
- 「（冷却）」中的「却」是复合词成分，不视为违规
- 替换后重读一遍确保语义流畅

## 修复模式 C：章节扩展（Chapter Expansion）

### 适用场景
补充/插章因 Phase 2 保守写策略或 delegate_task 超时未写完，字数不足目标 50%。

### 扩展原则（按重要性排序）
1. **不加新情节** — 不引入新的事件、新角色、新设定。扩展的是已有场景的表现密度
2. **感官细节优先** — 视觉→听觉→触觉→嗅觉→味觉，逐层补充
3. **内心戏次优先** — 角色在面对该场景时的心理活动、回忆闪回、潜意识的原始反应
4. **环境氛围最后** — 时间、天气、空间氛围，作为间隔填充

### 各类场景的扩展手法

| 场景类型 | 扩展手段 | 示例 |
|:---------|:---------|:-----|
| 独白/内心 | 增加回忆闪回、感官触发、自由联想 | 从「她觉得不真实」扩展到她穿越前最后一刻的画面 |
| 对话场景 | 加入动作标记、沉默的间隙、微表情变化 | 「他说好。」→「他嗯了一声，低头看着茶杯，转了半圈才放下。」 |
| 日常场景 | 增加环境声音、气味、氛围细节 | 胡同口的早点摊、远处小孩的笑声、空气中的葱油饼味 |
| 情感高潮 | 拉长时间轴——从事件前到事件后，不止写当时 | 拍照前的心态 + 拍照后的余韵 + 当晚的回想 |

### 字数目标设定
| 章节类型 | 建议目标 |
|:---------|:--------:|
| 插章/补充 | 1500-2000字 |
| 主章 | 2000-3000字 |
| 高潮/名场面章 | 2500-3500字 |

### 验证
```python
import re, glob
for f in sorted(glob.glob('chapters/ch_*.md')):
    with open(f) as fh:
        content = fh.read()
    zh = len(re.findall(r'[\u4e00-\u9fff\uf900-\ufaff]', content))
    if zh < 1000:
        print(f'⚠️ {f}: {zh}字（偏短）')
```

## 修复模式 D：章节压缩（Overshoot Compression）

### 适用场景
章节超长，超过目标字数的 30% 以上（如 5000 字的 target 写到了 6500+）。需要在不损失核心内容的前提下缩减篇幅。

### 核心策略
压缩不是删内容——是**提纯**：砍掉多余的修饰、重复的描景、过长的内心独白、低信息量的对话应对。

### 优先级排序（从最容易砍的开始）
| 优先级 | 砍什么 | 每处省多少 | 示例 |
|:-----:|:-------|:---------:|:-----|
| 🔴 P1 | **角色内心推理链条** | 50-200字 | 「一、我背后没人。二、铜镜里有张脸。……」→ 一句话带过 |
| 🔴 P2 | **背景设定回顾/早前蠢事** | 50-150字 | 「他这辈子做过很多蠢事。2018年P2P，2020年挖矿，2022年量子理疗仪……」→ 直接删除 |
| 🔴 P3 | **重复的鬼怪传说描写** | 30-100字 | 船上有鬼的多个证人说法→只留一句 |
| 🟡 P4 | **多余的短段分行** | 10-30字/处 | 连续短段→合并为一段，减少段落间距浪费 |
| 🟡 P5 | **修饰性副词/冗余句式** | 5-15字/处 | 「抓紧扶手」→「抓着」；「是像有人在床底下顶床板」→「有人顶床板」 |
| 🟢 P6 | **过渡性回应（对话中）** | 10-30字/处 | 「那你现在要去哪？」→直接进入话题，省略过渡 |

### 执行流程（迭代式）

**Step D1：摸底**
```python
import re
with open('chapters/ch_NN.md') as f:
    content = f.read()
zh = len(re.findall(r'[\u4e00-\u9fff\uf900-\ufaff]', content))
print(f'{zh}字, 目标{target}, 需砍{zh-target}字')
```

**Step D2：分批次压缩（每次砍 10-20% 的目标削减量）**
不要一次性砍完目标字数。理由是：每次 patch 后，上下文变化可能暴露出新的可缩点。

```python
# 批次策略：
# Batch 1: 砍 P1（推理链条/背景设定）  → 约150-300字
# Batch 2: 砍 P3（重复描写/冗余叙述） → 约100-200字
# Batch 3: 砍 P4（段落合并）           → 约50-100字
# Batch 4: 砍 P5（副词/冗余句式）      → 约50-100字
# 每轮之间：recount，判断是否已到目标
```

**Step D3：每轮压缩后检查**
```python
zh = len(re.findall(r'[\u4e00-\u9fff\uf900-\ufaff]', content))
print(f'{zh}字, 还需{zh - target}字')
if zh <= target:
    break
```

**Step D4：结束条件**
- 达到目标字数范围：✅ 停止
- 已达目标字数的 90% 且再压会损伤可读性：🟡 接受（不要过度压缩）
- 每轮削减量 < 20字 且距离目标仍有 10%+ 偏差：⚠️ 建议接受

### 常见坑

| 坑 | 症状 | 解决 |
|:---|:-----|:-----|
| **过度压缩** | 场景失去空间感，对话变成贴标签 | 设定"保底字数"——至少保留关键场景的节奏感 |
| **只砍大幅段落** | 小词重复累计可达 200-300 字 | 「忽然」「轻轻地」「慢吞吞地」等副词累计砍 |
| **对话压缩过头** | 角色失去说话风格 | 保留标志性的口头禅和语气节奏 |
| **patch 重复内容** | 之前的 patch 没完全清理，新旧内容并存 | patch 后用 `read_file` 检查该区域；如果发现重复，再 patch 删掉残留 |

### 字数日志模板
```
ch_NN 压缩日志:
  原字数: 6567
  P1 推理链条精简: → 删除80字 (6487)
  P2 背景设定删除: → 删除120字 (6367)
  P3 重复描写合并: → 删除60字 (6307)
  P4 段落合并:     → 删除40字 (6267)
  最终: 5798字 (减少769字/12%)
  状态: ✅ 接近目标
```

当修改简报中包含多个文风问题（通常是 P1+P2+P3 并发），按以下顺序执行：

```
Round 1: P1 缺失章节补写       → 确保完整性，这是后续所有修改的输入
Round 2: P2 加粗削减            → 最影响文风观感的违规
Round 3: P3 禁用词清零          → AI 味主要来源
Round 4: P4 章节扩展            → 篇幅均衡
```

每个 Round 内，用 delegate_task 并行处理。主 agent 负责：
1. 用户确认后一次性定义所有 Round 的工作范围
2. 每轮结束后做全量验证（不是等到全部做完）
3. 发现残留自动 patch，不留给用户反馈
