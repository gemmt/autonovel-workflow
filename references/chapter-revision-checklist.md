# 章节修改清单 · Chapter Revision Checklist

实战来源：枫火对《末日降临，末世游戏》第一章的7条修改意见。
每一条都是**可泛化的质量准则**，不限于该章或该小说。

---

## 1️⃣ 剧情逻辑 · Event Causality

**准则：** 关键事件的原因应避免过于偶然的物理损坏，优先选择系统/人为失误。

| 差 | 好 |
|:---|:---|
| 施工卡车吊臂刮到中继器，松了光纤 | 调度系统软件更新后缓存溢出bug，每个数据包校验标识错位 |

**检查：** 你的关键事件触发原因是「巧合」还是「系统漏洞」？巧合在现实中存在，但在叙事中会削弱逻辑可信度。

---

## 2️⃣ 对话密度 · Dialogue Compression

**准则：** 解释性对话（角色A向角色B解释背景）要压缩到最短。展示 > 解释。

**修复模式：** 用**即时演示**替代**口头解释**。
- 原：陈先生解释"不是CGI不是电影不是特效"
- 改：陈先生抬手让盒饭悬浮七秒

**检查清单：**
- [ ] 这段对话中，有没有信息可以用动作/演示替代？
- [ ] 对话双方的信息差是否合理？还是角色B只是在替读者问问题？
- [ ] 能否删掉角色的重复解释，让听者直接质疑并得到更高效的回应？

---

## 3️⃣ 性格前置 · Trait Early-Buildup

**准则：** 主角的核心性格特质应在第一个冲突场景中就有暗示，不要在后续谈判/决策时才"突然"展现。

**示例：** 冯枫的"算账"性格在地铁场景中已有内核算账（"载客车约200人，空车0人"），而不是等到签合同时才提到他会算。

**检查清单：**
- [ ] 主角的核心特征（理性/冲动/计算/直觉/善良/自私）是否在第一章前1/3就有具体行为支撑？
- [ ] 这些行为是否自然嵌入了场景，而不是插入式的"设定说明"？
- [ ] 后期的关键决策，前期的性格铺垫足够让读者觉得"果然如此"吗？

---

## 4️⃣ 场景过渡 · Scene Transition Arc

**准则：** 重大事件后的过渡不能突然跳转（场景A剧烈 → 场景B日常）。必须插入中间节奏：社会反应、舆论影响、角色对事件的再消化。

**结构公式：**
```
重大事件 → 社会/他人反应（媒体+同事+路人）→ 角色在平凡日常中反思 → 下一个事件敲门
```

**示例：** 地铁救人后 → 手机被打爆（3家电视台+2家报社+7个网红）→ 同事眼神变了、主管依然问"报告写完了吗" → 加班回家买盒饭、关掉未接来电 → 门铃响

---

## 5️⃣ 句子简洁 · Sentence Conciseness

**准则：** 角色"大脑运算/感知/判断"的描写，用最简方式呈现，不要逐步骤拆解。

| 差（22字，6个破折号，逐步骤枚举） | 好（16字，1句） |
|:---|:---|
| 他的大脑在零点三秒内完成了以下运算：看见了——两条灯是同一条轨道——方向和速度判断——左边的灯更大更宽，是载客列车——右边的灯更小更窄但速度更快—— | 零点三秒。他的大脑自动跑完了一笔算账：载客车大约两百人，空车零人。选项只有一个，不存在变量。 |

**规则：** 角色在高压下的认知，呈现结论而非过程。读者自己脑补"如何得出这个结论"比被带着走完运算过程更有代入感。

---

## 6️⃣ 词汇多样性 · Vocabulary Repetition

**准则：** 同一情感/动作词汇在同一场景中不应重复出现3次以上。

**高频易重复词清单（中文网文）：**
- 抖/发抖/颤抖（→ 战栗/哆嗦/颤动/晃动/痉挛）
- 突然/忽然（→ 可删，或改为"毫无征兆地""一瞬间"）
- 看着（→ 改为动作视角：转身/抬头/皱眉等）
- 觉得（→ 改为感官直接描述）
- 好像/仿佛（→ 删或改为比喻实体）

**修复脚本：**
```python
import re
repetitive = ['抖', '忽然', '突然', '猛地', '看着', '觉得']
for word in repetitive:
    count = len(re.findall(word, content))
    if count > 3:
        print(f'⚠️ "{word}" appears {count} times — consider replacement')
```

---

## 7️⃣ 场景描述效率 · Description Density

**准则：** 场景结尾/氛围描写的目标是传递核心信息（时间+空间+角色状态+目标），不要用诗意堆砌。

| 差（45字，3句，大量氛围堆砌） | 好（28字，2句，信息密度高） |
|:---|:---|
| 他蹲在那片锈红色的夕阳下，等着。夜幕正在从东方的废墟城市上空降下来。而那座工业园区里，有什么东西——正在苏醒。 | 他蹲在锈红色的夕阳下，等夜幕降临。眺望工业园区深处，目标在他视网膜上是一个小红点。 |

**检查：** 每段环境描写是否承担了至少两种功能？如果一段描写只传递"美/氛围"而不推进情节、不揭示角色、不提供信息，删掉它。

## 修改优先级

| 优先级 | 项 | 原因 |
|:-----:|:--|:-----|
| 🔴 P0 | ① 剧情逻辑 | 硬伤级，必须修复 |
| 🔴 P0 | ② 对话密度 | 读者会直接跳过 |
| 🟡 P1 | ③ 性格前置 + ④ 过渡 | 角色说服力，节奏感 |
| 🟡 P1 | ⑤ 句子简洁 | 可读性，不影响剧情 |
| 🟢 P2 | ⑥ 词汇多样性 | 打磨性质 |
| 🟢 P2 | ⑦ 描述效率 | 打磨性质 |

## 应用场景

- **Phase 3a 对抗式编辑：** 作为追加维度，在37维审计之外检查这7项
- **Phase 3b 深度修复循环：** 每轮修复后逐项核对
- **自评/章节门控：** 第二章及以后的初稿可直接用此清单做前置检查
